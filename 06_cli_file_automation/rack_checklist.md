# [교육용 샘플] 랙 현장 확인 체크리스트 (EDU-RACK-CHECKLIST)

> **※ 본 문서는 AWS DCO 인턴십 직무 이해를 위한 교육용 샘플 데이터를 사용하고 있으며, 실제 운영 환경 및 실제 AWS 서비스의 데이터가 아닙니다.**

---

## 1. 개요 및 목적
본 체크리스트는 발생한 장애 티켓(`sample_ticket.md` 등) 및 장비 로그 분석 결과를 검증하고, 실제 현장(데이터 센터 랙) 점검 작업을 수행하기 전 단계에서 필요한 확인 사항을 체계적으로 기록하기 위한 교육용 양식입니다.

---

## 2. 체크리스트 항목

### [ ] 항목 1: 발생 시간 확인
* **확인 내용**: 티켓에 기록된 장애 발생 시점(`Timestamp`)과 관련 시스템 로그 타임스탬프가 일치하는지 대조합니다.
* **기록**: 
  - 티켓 발생 시간: `2026-07-07 15:00:00 KST`
  - 로그 상 시간 일치 여부: [ ] 예 / [ ] 아니오

### [ ] 항목 2: 샘플 장비명 확인
* **확인 내용**: 문제가 발생한 가상 장비 식별자가 올바른지 확인하고, 해당 랙(RACK)의 위치 정보를 기록합니다. (실제 데이터 대신 SAMPLE 명칭 사용)
* **기록**:
  - 장비명: `SAMPLE_TOR_SW_01`
  - 랙 위치 (가상): `EDU-RACK-A01`

### [ ] 항목 3: 장애 반복 여부 확인
* **확인 내용**: 해당 포트나 장비에서 최근 24시간 이내에 유사한 CRC 에러 또는 링크 다운/업(Flapping) 이력이 있었는지 이전 장애 티켓 이력을 조회합니다.
* **기록**:
  - 반복성 여부: [ ] 최초 발생 / [ ] 간헐적 발생 / [ ] 지속적 발생

### [ ] 항목 4: Ticket 상태 확인
* **확인 내용**: 현재 관리 티켓의 상태가 정당하게 이관(Escalate)되거나 접수 상태인지 파악합니다.
* **기록**:
  - 티켓 상태: [ ] Open / [ ] In-Progress / [ ] Escalated

### [ ] 항목 5: 원인 단정 금지 원칙 준수
* **확인 내용**: 
  - 링크 다운의 직접적인 원인을 현장 확인 전에 단순 케이블 불량으로 단정 짓지 않습니다.
  - **주의**: 포트 광 트랜시버(Transceiver) 불량, 상위 패치 패널 포트 먼지, 혹은 스위치 ASIC 모듈 자체의 장애 가능성을 모두 열어두고 현장 점검용 자재(예: 대체용 광케이블 및 트랜시버)를 준비해야 합니다.
  - 서명/동의 여부: [ ] 확인 완료

## 3. 보고서 초안 작성 메모

* 현장 점검 이후 결과 보고서 작성을 위해 아래 사항들을 로그 및 티켓 분석 단계에서 미리 메모해 둡니다.
  ```text
  1. 이벤트 인지 단계: (예: CRC 에러 누적 관찰 시점 및 Link Down 발생 시점)
  2. 현장 확인 시 준비할 스페어 부품: (예: SAMPLE_TOR_SW_01의 Ethernet1/1에 맞는 가상 규격 케이블 등)
  3. 교체 작업 후 검증 계획: (예: 장비 내 Loopback 상태 검사 및 포트 에러 카운터 초기화 후 모니터링)
  ```

## 4. 보안 가이드라인
* **외부 대상 점검 명령(예: 실제 IP 대상 ping, traceroute 등)을 외부 네트워크로 수행하거나 이를 지침에 명시해서는 안 됩니다.**
* 모든 확인 작업은 로컬 모의 터미널 상에서 모의 출력 결과를 읽고 기록하는 범주 내로 한정합니다.
* 실제 시리얼 번호, 실제 고객의 서비스 트래픽 경로 등의 민감 정보는 어떠한 필드에도 입력하지 않습니다.
