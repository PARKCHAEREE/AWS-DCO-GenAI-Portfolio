## 2. 내가 만든 DCO 인프라 용어 정의

### Rack
내 정의: 서버 같은 장비를 여러 층으로 쌓아 세워두는 장비용 선반이다.

### Server
내 정의: 데이터를 저장하고 계산을 처리하는 컴퓨터로, 많은 요청을 동시에 처리할 수 있게 만들어진 장비이다.

### NIC
내 정의: 서버가 네트워크에 연결되어 데이터를 주고받을 수 있게 해주는 부품이다.

### ToR Switch
내 정의: 한 Rack 안의 여러 서버를 묶어 네트워크에 연결해주는 스위치 장치이다.

### Link Down
내 정의: 네트워크 연결이 끊어졌음을 알려주는 신호로, 정확한 원인은 별도 확인이 필요하다.

### CRC error
내 정의: 데이터가 전달되는 과정에서 손상되었을 가능성을 알려주는 오류 신호로, 원인은 여러 가지일 수 있다.

### SSD failure
내 정의: 데이터를 저장하는 부품이 정상적으로 작동하지 않는 상태를 나타내는 신호로, 정확한 원인은 확인이 필요하다.

### Fan Alert
내 정의: 장비를 식혀주는 냉각 팬에 이상이 있을 수 있음을 알리는 신호로, 원인은 단정할 수 없다.

## 3. 헷갈리는 용어 비교
**내가 비교한 용어:** SSD failure vs Fan Alert

**내가 이해한 차이:** SSD failure는 데이터를 저장하는 부품 자체의 문제이고, Fan Alert는 장비를 식혀주는 냉각 부품의 문제입니다. 하나는 "저장" 쪽 문제, 다른 하나는 "냉각" 쪽 문제라서 다루는 영역이 다릅니다.

**아직 헷갈리는 부분:** 두 문제가 서로 완전히 무관한 건 아니라고 배웠는데(냉각이 안 되면 다른 부품에도 영향을 줄 수 있다는 점), 실제로 어느 정도까지 연관되는지는 아직 명확히 감이 안 옵니다.

## 4. 앞으로 사용할 학습 보조 프롬프트

### 프롬프트 1. 모르는 DCO 용어 설명용
**역할:** 데이터센터 운영 직무를 처음 배우는 학생을 돕는 학습 도우미
**목표:** DCO 관련 용어를 비전공자도 이해할 수 있게 쉽게 설명해달라고 요청
**상황:** AWS DCO 인턴십 직무 이해를 위한 생성형 AI 활용 수업 수강 중이며, 공개 직무 맥락과 교육용 예시로만 설명받기
**출력 형식:** 쉬운 설명 / DCO 직무 이해에서 나올 수 있는 상황 / 내 말로 바꿔 쓸 표현 / 주의할 점
**제한 조건:** 실제 내부 절차처럼 단정하지 않기, 실제 장비명·티켓 번호·계정정보 금지, 예시는 SAMPLE·DEMO·EDU 표현만 사용

### 프롬프트 2. 비슷한 용어 비교용
**역할:** DCO 인프라 용어를 쉽게 비교해주는 학습 도우미
**목표:** 헷갈리는 두 용어의 차이를 비전공자도 이해할 수 있게 설명해달라고 요청
**상황:** 두 용어가 비슷하게 느껴져서 구분하는 연습 중이며, 공개 직무 맥락과 교육용 예시 기준으로 설명받기
**출력 형식:** 용어 A 의미 / 용어 B 의미 / 가장 큰 차이 / 교육용 예시 / 기억법 / 주의할 점
**제한 조건:** 실제 내부 절차·장비명·티켓 번호·고객정보 금지, 원인을 하나로 단정하지 않기

### 프롬프트 3. 장애 표현 확인 순서 정리용
**역할:** DCO 직무의 장애 대응 흐름을 학생이 이해하도록 돕는 학습 도우미
**목표:** 장애 표현을 보고 가능성 있는 원인들과 확인 순서를 교육용으로 정리해달라고 요청
**입력:** 장애 관련 알림/증상 문장 (예: "Link Down 알림이 떴다")
**출력 형식:** 장애 표현의 쉬운 의미 / 가능성 있는 원인들 / 확인해볼 수 있는 순서 / 관련 DCO 용어 / 주의할 점
**제한 조건:** 원인을 하나로 단정하지 않기, 실제 내부 절차·장비명·티켓 번호·계정정보 금지, "즉시 조치 가능"·"정확한 원인입니다" 같은 단정 표현 금지

## 5. AI 답변을 보고 내가 수정한 점
**어려워서 쉽게 바꾼 표현:** "Cyclic Redundancy Check error" 같은 정식 명칭 대신, "데이터가 전달되는 도중 깨졌다는 걸 알려주는 오류 신호"처럼 풀어서 이해했습니다.

**실제 정보처럼 보일 수 있어 삭제하거나 수정한 표현:** 예시에 나온 `SSD-DEMO-01`, `Fan-DEMO-02` 같은 표기가 실제 장비 코드처럼 보이지 않도록, 모두 교육용 가상 표기라는 점을 다시 확인했습니다.

**너무 단정적이라 바꾼 표현:** "SSD failure는 부품 고장이다" 식으로 단정하기보다, "정상적으로 작동하지 않는 상태를 나타내는 신호이며 원인은 확인이 필요하다"는 식으로 바꿨습니다.

**아직 헷갈리는 용어:** CRC error와 Link Down의 차이 (둘 다 네트워크 관련 오류라서 구분이 아직 헷갈립니다.)

**다시 공부하고 싶은 용어:** ToR Switch — 실제로 어떤 구조로 여러 서버와 연결되는지 좀 더 알고 싶습니다.

## 6. 오늘의 느낀 점
**오늘 처음 알게 된 용어는:** Fan Alert, SSD failure

**가장 헷갈렸던 용어는:** SSD failure와 Fan Alert — 둘 다 "장애 알림"이라는 점은 같아서 처음엔 비슷하게 느껴졌습니다.

**AI 답변을 사용할 때 조심해야 할 점은:** AI가 원인을 편하게 단정해서 설명할 수 있는데, 실제로는 여러 가능성을 열어두고 확인하는 태도가 필요하다는 것을 느꼈습니다.

**다음 차시에서 더 알고 싶은 내용은:** 실제 장애 알림이 발생했을 때 Ticket을 어떻게 작성하고, 어떤 기준으로 Escalation까지 이어지는지 흐름을 더 알아보고 싶습니다.
